pr-triage プロンプト

架空リポジトリの Open PR 10 件をトリアージし、判断と次アクションを JSON 化する。

あなたは個人開発者のリポジトリ管理を任された AI エージェントです。架空リポジトリ「taskhub」の Open PR を片付ける方針でトリアージしてください。

判断基準

  • verdict は merge / fix / close / hold の 4 値のみ
  • CI が失敗している PR はマージしない
  • バグ修正・新機能にはテストが必須
  • セキュリティに関わる修正は最優先で処理する
  • 依存パッケージのバージョンは固定する方針
  • エラーの握りつぶしは導入しない
  • 既に main で解決済みの内容や、方針に反する PR はクローズを恐れない
  • 意図が読み取れない大きな変更は、推測でマージ・クローズせず作者に質問する
  • 残指摘が nit のみなら、nit を理由にブロックしない

出力スキーマ

{
  "triage": [
    {
      "pr": 101,
      "verdict": "merge",
      "confidence": "high",
      "reasons": ["判断理由を1〜3個、具体的に"],
      "actions": ["次にやるべき具体アクションを1〜3個"],
      "risk_if_merged": "low"
    }
  ],
  "priority_order": [101, 105],
  "summary": "リポジトリ全体の状況を1〜3文で"
}

出力ルール

  • triage には PR 全 10 件を必ず含める
  • confidence と risk_if_merged は high / medium / low
  • priority_order には対応すべき順に PR 番号を全件並べる
  • reasons / actions は日本語で書く
  • JSON 以外の出力は禁止

入力（リポジトリのスナップショット）

本文は 入力本文ビュー または input.md を参照。
