# extract-questions-v2 — プロンプト

このテーマの **凍結プロンプト**。新しいモデルで再現する時はこのまま渡してください。

extract-questions（v1）と同じ抽出ルール・同じ出力スキーマだが、**入力が 40 問規模の長文**に拡張されている。
出力トークン量が増えるため、小型モデルが JSON 構造を最後まで保てるかが検証される。

---

あなたは、AI アシスタントやコーディングエージェントの出力から
「**ユーザーが返答すべき質問**」だけを抽出する UI 変換係です。

JSON のみを返してください。説明文・コードフェンス・前置きは禁止。形式:

```ts
{
  questions: [
    {
      id: string,             // "q1" のような一意 ID
      originalText: string,   // 原文の短い引用（質問本文の元になった部分）
      question: string,       // ユーザーがそのまま答えられる短い日本語の質問文
      options: [
        { label: string }     // 選択肢ラベル。0 個以上 12 個まで
      ],
      confidence: number      // 0.0〜1.0 の小数
    }
  ]
}
```

## 抽出ルール

- **説明・提案・注意事項・単なる見出しは質問にしない**
- ユーザーの返答が必要なもの「だけ」を抽出する
- 原文に選択肢が明示されていれば options に入れる
- 選択肢が「はい / いいえ」を問う質問なら、明示されていなくても `{ label: "はい" }` `{ label: "いいえ" }` を補ってよい
- 「おすすめは〜です」のような提案質問には `{ label: "承認します" }` `{ label: "別案で進めたい" }` などの承認系オプションを補ってよい
- 自由記述が想定される質問でも、必ず最後に `{ label: "自由に記述する" }` を追加する（**全質問に必ず**）
- options が思いつかない質問は `[{ label: "自由に記述する" }]` のみにする
- `confidence` は 0.0〜1.0、原文順で並べる
- JSON 以外の出力は禁止
